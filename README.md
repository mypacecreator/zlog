# zlog

超軽量・高速なロギングCLIツール

作業時間を記録するためのシンプルなコマンドラインツールです。案件名と作業内容をコロン（`:`）で区切って入力するだけで、CSVファイルに自動記録されます。

## 特徴

- **高速**: 0.1秒以内にログ記録完了
- **シンプル**: クォート不要で自然に入力可能
- **柔軟**: 1桁時刻、全角デリミタにも対応
- **自動補完**: 前タスクの自動終了、時刻の自動0埋め

## インストール

```bash
cd /Users/kei/projects/_zlog
npm install
volta run npm link
```

## 基本的な使い方

### 記法

```
zlog [時刻] 案件名:作業内容
zlog [時刻] 作業内容のみ
zlog [時刻] /
zlog [時刻] / 案件名:作業内容
```

### デリミタ

案件名と作業内容を区切る記号。**すべてクォート不要**で使えます。

| デリミタ | 種類 | 例 |
|---------|------|-----|
| `:` | 半角コロン（推奨） | `zlog 案件A:作業内容` |
| `：` | 全角コロン | `zlog 案件A：作業内容` |
| `｜` | 全角パイプ | `zlog 案件A｜作業内容` |

### 時刻指定

時刻は省略可能（現在時刻を自動使用）。指定する場合は1桁・2桁両対応。

```bash
zlog 案件A:作業開始          # 現在時刻で記録
zlog 9:00 案件A:作業開始     # 09:00で記録（自動0埋め）
zlog 14:30 案件B:ミーティング # 14:30で記録
zlog 8:5 案件C:実装          # 08:05で記録（自動0埋め）
```

## ロギングパターン

### パターン① 開始

新しいタスクを開始します。未終了タスクがあれば自動的に終了させます。

```bash
# 案件名と作業内容を記録
zlog プロジェクトA:資料作成

# 作業内容のみ記録（案件名省略）
zlog 休憩
zlog ミーティング

# 時刻を指定
zlog 10:00 プロジェクトB:実装作業
```

**CSV記録例:**
```csv
2026-04-26,10:30,,プロジェクトA,資料作成
2026-04-26,11:00,,休憩
```

### パターン② 終了

進行中のタスクを終了します。

```bash
# 現在時刻で終了
zlog /

# 指定時刻で終了
zlog 12:00 /
```

未終了タスクがない場合はエラーメッセージが表示されます。

```bash
$ zlog /
エラー: 終了すべきタスクがありません
```

**CSV記録例（終了後）:**
```csv
2026-04-26,10:30,11:45,プロジェクトA,資料作成
```

### パターン③ 連続

前回タスクの終了時刻から、新しいタスクを開始して即座に終了します。

```bash
# 前回の終了時刻〜現在時刻まで記録
zlog / ランチ

# 前回の終了時刻〜指定時刻まで記録
zlog 13:00 / ランチ
```

**使用条件:**
- 未終了タスクがないこと
- 前回に完了したタスクが存在すること

**CSV記録例:**
```csv
2026-04-26,10:00,11:00,プロジェクトA,資料作成
2026-04-26,11:00,12:00,ランチ              # 連続実行
```

## 実用例

### 一日の作業記録

```bash
# 09:00 業務開始
zlog 9:00 プロジェクトA:設計書作成

# 10:30 タスク切り替え（前タスクは自動終了）
zlog プロジェクトB:実装作業

# 12:00 昼休憩（案件名省略）
zlog 12:00 /
zlog / ランチ

# 13:00 午後の作業開始
zlog 13:00 プロジェクトC:レビュー対応

# 15:00 会議
zlog 15:00 /
zlog / 定例ミーティング

# 17:00 作業再開
zlog 17:00 プロジェクトA:実装

# 18:30 終業
zlog 18:30 /
```

**記録されるCSV:**
```csv
2026-04-26,09:00,10:30,プロジェクトA,設計書作成
2026-04-26,10:30,12:00,プロジェクトB,実装作業
2026-04-26,12:00,13:00,,ランチ
2026-04-26,13:00,15:00,プロジェクトC,レビュー対応
2026-04-26,15:00,17:00,,定例ミーティング
2026-04-26,17:00,18:30,プロジェクトA,実装
```

### 後からまとめて記録

```bash
# 午前の作業をまとめて記録
zlog 9:00 案件A:要件定義
zlog 11:00 /

# 午後の作業をまとめて記録
zlog 13:00 案件B:開発
zlog 16:00 /
zlog / 案件C:テスト
zlog 18:00 /
```

## ログファイル

### 保存場所

```
/Users/kei/projects/_zlog/logs/daily_log.csv
```

### CSVフォーマット

ヘッダー行なし、5フィールド構成：

```
日付(YYYY-MM-DD),開始時刻(HH:MM),終了時刻(HH:MM),案件名,作業内容
```

- 終了時刻が未確定の場合は空文字列
- 案件名が省略された場合は空文字列

### ログの確認

```bash
# ログファイルを表示
cat /Users/kei/projects/_zlog/logs/daily_log.csv

# または（プロジェクトディレクトリから）
cat logs/daily_log.csv
```

## トラブルシューティング

### コマンドが見つからない

```bash
# Voltaのシムが作成されていない場合
volta run npm link

# コマンドキャッシュをクリア
hash -r

# 新しいシェルセッションで確認
which zlog
zlog --version
```

### 時刻が正しく認識されない

時刻は `H:MM` または `HH:MM` 形式で入力してください。

```bash
# OK
zlog 9:00 案件A:作業
zlog 09:00 案件A:作業
zlog 14:30 案件B:作業

# NG
zlog 9 案件A:作業        # 分が必要
zlog 9:0 案件A:作業      # 分は2桁で
```

### デリミタを忘れた場合

デリミタなしでも動作しますが、作業内容のみ記録されます（案件名は空）。

```bash
zlog 作業内容だけ
# → 案件名: 空、作業内容: 作業内容だけ
```

案件名も記録したい場合は、デリミタ（`:`、`：`、`｜`）を使ってください。
