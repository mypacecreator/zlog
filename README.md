# zlog

超軽量・高速なロギングCLIツール

作業時間を記録するためのシンプルなコマンドラインツールです。案件名と作業内容をコロン（`:`）で区切って入力するだけで、CSVファイルに自動記録されます。

## 特徴

- **高速**: 0.1秒以内にログ記録完了
- **シンプル**: クォート不要で自然に入力可能
- **柔軟**: 1桁時刻、全角デリミタにも対応
- **自動補完**: 前タスクの自動終了、時刻の自動0埋め
- **AI要約**: `zlog summarize` で当日ログをAIがカテゴリ付与・整形して月別アーカイブに保存

## インストール

```bash
cd /Users/kei/projects/_zlog
npm install
volta run npm link
```

## 基本的な使い方

### 記法

```
zlog [日付] [時刻] 案件名:作業内容
zlog [日付] [時刻] 作業内容のみ
zlog [日付] [時刻] /
zlog [日付] [時刻] / 案件名:作業内容
```

- `[日付]`: YYYY-MM-DD形式（省略時は今日）
- `[時刻]`: H:MM または HH:MM形式（省略時は現在時刻）

### デリミタ

案件名と作業内容を区切る記号。**すべてクォート不要**で使えます。

| デリミタ | 種類 | 例 |
|---------|------|-----|
| `:` | 半角コロン（推奨） | `zlog 案件A:作業内容` |
| `：` | 全角コロン | `zlog 案件A：作業内容` |
| `｜` | 全角パイプ | `zlog 案件A｜作業内容` |

### 日付・時刻指定

日付・時刻は省略可能（今日の現在時刻を自動使用）。指定する場合は1桁・2桁両対応で自動0埋め。

```bash
# 日付・時刻なし
zlog 案件A:作業開始

# 時刻のみ指定
zlog 9:00 案件A:作業開始     # 今日の09:00で記録
zlog 8:5 案件B:実装          # 今日の08:05で記録（自動0埋め）

# 日付と時刻を指定
zlog 2026-04-25 14:30 案件C:ミーティング  # 2026-04-25 14:30で記録
zlog 2026-4-5 9:00 案件D:過去ログ        # 2026-04-05 09:00で記録（自動0埋め）

# 日付のみ指定
zlog 2026-04-25 案件E:作業   # 2026-04-25の現在時刻で記録
```

## ロギングパターン

### パターン① 開始

新しいタスクを開始します。未終了タスクがあれば自動的に終了させます。

```bash
# 案件名と作業内容を記録
zlog プロジェクトA:資料作成

# 作業内容のみ記録（案件名省略）
zlog 休憩
zlog ミーティング

# 時刻を指定
zlog 10:00 プロジェクトB:実装作業
```

**CSV記録例:**
```csv
2026-04-26,10:30,,プロジェクトA,資料作成
2026-04-26,11:00,,休憩
```

### パターン② 終了

進行中のタスクを終了します。

```bash
# 現在時刻で終了
zlog /

# 指定時刻で終了
zlog 12:00 /
```

未終了タスクがない場合はエラーメッセージが表示されます。

```bash
$ zlog /
エラー: 終了すべきタスクがありません
```

**CSV記録例（終了後）:**
```csv
2026-04-26,10:30,11:45,プロジェクトA,資料作成
```

### パターン③ 連続

前回タスクの終了時刻から、新しいタスクを開始して即座に終了します。

```bash
# 前回の終了時刻〜現在時刻まで記録
zlog / ランチ

# 前回の終了時刻〜指定時刻まで記録
zlog 13:00 / ランチ
```

**使用条件:**
- 未終了タスクがないこと
- 前回に完了したタスクが存在すること

**CSV記録例:**
```csv
2026-04-26,10:00,11:00,プロジェクトA,資料作成
2026-04-26,11:00,12:00,ランチ              # 連続実行
```

## 実用例

### 一日の作業記録

```bash
# 09:00 業務開始
zlog 9:00 プロジェクトA:設計書作成

# 10:30 タスク切り替え（前タスクは自動終了）
zlog プロジェクトB:実装作業

# 12:00 昼休憩（案件名省略）
zlog 12:00 /
zlog / ランチ

# 13:00 午後の作業開始
zlog 13:00 プロジェクトC:レビュー対応

# 15:00 会議
zlog 15:00 /
zlog / 定例ミーティング

# 17:00 作業再開
zlog 17:00 プロジェクトA:実装

# 18:30 終業
zlog 18:30 /
```

**記録されるCSV:**
```csv
2026-04-26,09:00,10:30,プロジェクトA,設計書作成
2026-04-26,10:30,12:00,プロジェクトB,実装作業
2026-04-26,12:00,13:00,,ランチ
2026-04-26,13:00,15:00,プロジェクトC,レビュー対応
2026-04-26,15:00,17:00,,定例ミーティング
2026-04-26,17:00,18:30,プロジェクトA,実装
```

### 後からまとめて記録

**今日の作業をまとめて記録:**
```bash
# 午前の作業をまとめて記録
zlog 9:00 案件A:要件定義
zlog 11:00 /

# 午後の作業をまとめて記録
zlog 13:00 案件B:開発
zlog 16:00 /
zlog / 案件C:テスト
zlog 18:00 /
```

**前日など過去の作業を記録:**
```bash
# 2026-04-25の作業を記録
zlog 2026-04-25 9:00 プロジェクトX:設計
zlog 2026-04-25 12:00 /
zlog 2026-04-25 / ランチ
zlog 2026-04-25 14:00 /
zlog 2026-04-25 14:00 プロジェクトY:実装
zlog 2026-04-25 18:00 /
```

**連続実行での日付またぎについて:**

連続実行（`zlog / 案件:作業`）では、前回タスクと同じ日付のみ指定可能です。日付をまたぐ場合は別々に記録してください。

```bash
# NG: 日付またぎはエラー
zlog 2026-04-25 23:00 案件A:深夜作業
zlog 2026-04-25 23:30 /
zlog 2026-04-26 / 案件A:深夜作業続き  # エラー！

# OK: 別々に記録
zlog 2026-04-25 23:00 案件A:深夜作業
zlog 2026-04-25 23:30 /
zlog 2026-04-26 0:00 案件A:深夜作業続き
zlog 2026-04-26 1:00 /
```

## AI要約・アーカイブ

業務終了後に `zlog summarize` を実行すると、当日のCSVログをAI（Anthropic API）でカテゴリ付与・整形し、月別アーカイブファイルへ自動追記します。

### 基本的な使い方

```bash
# 今日のログを要約してアーカイブ
zlog summarize

# エイリアス
zlog s

# 日付を指定して要約（過去日の整理に）
zlog summarize --date 2026-04-25
zlog s -d 2026-04-25
```

### 出力フォーマット

1ログにつき1行で出力されます。

```
[2026-04-26] 09:00-10:30 | プロジェクトA | 設計書作成 | A2📋 | 所要：90分
[2026-04-26] 10:30-12:00 | プロジェクトB | 実装作業 | A3🔨 | 所要：90分
[2026-04-26] 12:00-13:00 |  | ランチ | D1☕ | 所要：60分
```

### アーカイブファイル

要約結果は月別MDファイルに自動追記されます。

```
logs/2026-04.csv         # ← ロギング元データ（変更なし）
archives/2026-04.md      # ← 要約・アーカイブ先
```

`archives/YYYY-MM.md` の構成：

```markdown
## 2026-04-26: 実績報告
[2026-04-26] 09:00-10:30 | プロジェクトA | 実装作業 | A1. 実作業 🛠️ | 所要：90分
[2026-04-26] 10:30-12:00 | プロジェクトB | ミーティング | A3. コンサルティング／打合せ 🧭 | 所要：90分

## 2026-04-27: 実績報告
[2026-04-27] 09:00-11:00 | | 経理処理 | B1. 事務系 📑 | 所要：120分
```

処理完了後、アーカイブしたセクションの内容がターミナルに出力されます（Gemini等への貼り付け用）。

### 事前準備

Anthropic APIキーを設定してください。

```bash
# 設定ファイルに保存（一度だけ）
zlog config set api_key sk-ant-xxxxxxxx

# または環境変数で指定
export ANTHROPIC_API_KEY=sk-ant-xxxxxxxx
```

### カテゴリ体系のカスタマイズ

`config/categories.md` を直接編集するとカテゴリ定義を変更できます。

デフォルトのカテゴリ：

| カテゴリ | 内容 |
|---------|------|
| A1〜A6 | クライアントワーク（打ち合わせ・制作・納品など） |
| B1〜B4 | 自社業務（社内MTG・営業・管理・自社開発） |
| C1〜C3 | 社会・学習（学習・外部イベント・社会貢献） |
| D1〜D2 | 私用・休憩（休憩・食事・雑務） |
| E0 | その他（分類不能） |

### AIプロンプトのカスタマイズ（上級）

`config/prompt.md` を作成すると、AIへの指示全体をカスタマイズできます。`{CATEGORIES}` と `{LOGS}` のプレースホルダーを使ってください。

```markdown
カテゴリ体系:
{CATEGORIES}

以下のログを整形してください:
{LOGS}
```

## 設定

`zlog config` コマンドで各種設定を管理します。

```bash
# 全設定を確認
zlog config list

# 設定を変更
zlog config set model claude-3-5-haiku-latest
zlog config set api_key sk-ant-xxxxxxxx

# 設定を取得
zlog config get model
zlog config get api_key
```

| 設定キー | 内容 | デフォルト |
|---------|------|-----------|
| `model` | 使用するAnthropicモデル | `claude-haiku-4-5` |
| `api_key` | Anthropic APIキー | なし（環境変数 `ANTHROPIC_API_KEY` も可） |

設定は `config.json` に保存されます（APIキー保護のため `.gitignore` 対象）。

## ログファイル

### 保存場所

ログは**月ごとに分割**されたファイルに保存されます。

```
/Users/kei/projects/_zlog/logs/2026-04.csv  # 2026年4月
/Users/kei/projects/_zlog/logs/2026-05.csv  # 2026年5月
/Users/kei/projects/_zlog/logs/2026-06.csv  # 2026年6月
```

ファイル名形式: `YYYY-MM.csv`

### CSVフォーマット

ヘッダー行なし、5フィールド構成：

```
日付(YYYY-MM-DD),開始時刻(HH:MM),終了時刻(HH:MM),案件名,作業内容
```

- 終了時刻が未確定の場合は空文字列
- 案件名が省略された場合は空文字列
- 月ごとにファイルが自動作成される（肥大化を防止）

### ログの確認

```bash
# 今月のログを表示
cat /Users/kei/projects/_zlog/logs/2026-04.csv

# 特定の月のログを表示
cat /Users/kei/projects/_zlog/logs/2026-03.csv

# または（プロジェクトディレクトリから）
cat logs/2026-04.csv

# すべての月のログを表示
cat logs/*.csv
```

## トラブルシューティング

### コマンドが見つからない

```bash
# Voltaのシムが作成されていない場合
volta run npm link

# コマンドキャッシュをクリア
hash -r

# 新しいシェルセッションで確認
which zlog
zlog --version
```

### 時刻が正しく認識されない

時刻は `H:MM` または `HH:MM` 形式で入力してください。

```bash
# OK
zlog 9:00 案件A:作業
zlog 09:00 案件A:作業
zlog 14:30 案件B:作業

# NG
zlog 9 案件A:作業        # 分が必要
zlog 9:0 案件A:作業      # 分は2桁で
```

### デリミタを忘れた場合

デリミタなしでも動作しますが、作業内容のみ記録されます（案件名は空）。

```bash
zlog 作業内容だけ
# → 案件名: 空、作業内容: 作業内容だけ
```

案件名も記録したい場合は、デリミタ（`:`、`：`、`｜`）を使ってください。

### 日付指定と連続実行

日付は `YYYY-MM-DD` 形式で入力してください。1桁の月・日は自動的に0埋めされます。

```bash
# OK
zlog 2026-04-25 案件:作業
zlog 2026-4-5 案件:作業      # 2026-04-05に正規化

# NG
zlog 2026/04/25 案件:作業    # スラッシュ不可
zlog 04-25 案件:作業         # 年が必要
```

**連続実行で「日付またぎはできません」エラーが出る場合:**

連続実行（`zlog / 案件:作業`）は、前回タスクと同じ日付のみ指定可能です。

```bash
# エラーが出る場合は、別々に記録してください
zlog 2026-04-25 23:00 案件A:作業
zlog 2026-04-25 23:30 /
zlog 2026-04-26 0:00 案件A:続き  # 連続実行を使わず、新規開始で記録
zlog 2026-04-26 1:00 /
```

### 月またぎと未終了タスク

ログは月ごとにファイルが分割されます。月をまたいだ操作には制限があります。

**「未終了タスクの日付が異なります」エラーが出る場合:**

同月内でも、未終了タスクと異なる日付で新規タスクを開始できません。

```bash
# 例: 4月30日に未終了タスクがある状態で、4月26日のタスクを追加
zlog 2026-04-30 23:00 案件A:月末作業
zlog 2026-04-26 案件B:別日の作業  # エラー！

# 解決方法: 先に未終了タスクを終了させる
zlog 2026-04-30 23:30 /           # 未終了タスクを終了
zlog 2026-04-26 案件B:別日の作業  # その後、別日のタスクを記録
```

**月初に前月の未終了タスクがある場合:**

月をまたいで未終了タスクがあっても、新しい月のタスクは記録できます。ただし、前月の未終了タスクは前月のファイルに残ります。

```bash
# 4月のタスクが未終了のまま、5月のタスクを記録
zlog 2026-04-30 23:00 案件A:4月のタスク  # 未終了
zlog 2026-05-01 案件B:5月のタスク       # 記録可能

# 前月の未終了タスクを終了する場合は、日付を指定
zlog 2026-04-30 23:30 /
```

**推奨運用:**
- 月末には未終了タスクを必ず終了させる
- 月初に前月のログファイルを確認して、未終了タスクがないかチェックする

```bash
# 前月のログを確認
cat logs/2026-04.csv | grep ",,"  # 終了時刻が空（未終了）の行を検索
```
