
# 雑ログ記録CLIツール『zlog』実装指示書

## 1. プロジェクト概要
日中の「思考のリズム」を止めないための、超軽量・高速なロギングCLIツール。AI（LLM）を介さずローカルで完結させ、業務終了後にAIで「カテゴリ付与・整理」を行うための正確な生データ（CSV）を蓄積することを目的とする。

## 2. 技術要件
 * **実行環境:** Node.js (npm)
 * **パッケージ管理:** package.json
 * **配布方法:** bin 設定により zlog コマンドとしてグローバルに実行可能にすること
 * **依存ライブラリ:** 必要最小限（commander 等）
 * **動作速度:** 0.1秒以内（API通信厳禁）

## 3. ロギング・ロジック（3つのパターン）
現在時刻をシステムから自動取得することを基本とするが、引数先頭に HH:MM 形式がある場合はその時刻を優先する（バックデート対応）。

| モード      | 入力コマンド例              | 処理内容                                                   |
| -------- | -------------------- | ------------------------------------------------------ |
| **① 開始** | zlog [時刻](#) 案件｜内容   | 現在進行中のタスクがあれば終了時刻を埋めて閉じ、新しい行を開始時刻と共に作成。                |
| **② 終了** | zlog [時刻](#) /       | 未終了タスクの終了時刻を埋める。未終了タスクがない場合はエラー。                       |
| **③ 連続** | zlog [時刻](#) / 案件｜内容 | **直前タスクの終了時刻を開始時刻として再利用**し、現在時刻（または指定時刻）を終了時刻として新行を作成。 |

## 4. データ仕様
### 保存先
~/zlog/daily\_log.csv （ディレクトリがない場合は作成すること）
### CSVフォーマット
一切の装飾・ヘッダーなしで以下を追記する。
日付(YYYY-MM-DD),開始時刻(HH:MM),終了時刻(HH:MM),案件名,作業内容

## 5. コマンドライン仕様の詳細
 * **時刻指定の優先:** 入力引数の第一要素が ^([01](#)d|2[0-3](#)):([0-5](#)d)$ にマッチする場合、その時刻を 開始/終了時刻 として採用し、残りの引数を 案件名｜内容 として扱う。
 * **デリミタ:** 案件名と作業内容の区切りには全角・半角の ｜ | を許容する。
 * **エラー処理:**
   * パターン②で未終了タスクがない場合：「終了すべきタスクがありません」
   * パターン③で参照すべき前回の終了時刻がない場合：「開始時刻を指定してください」
   \* 
## 6. 実装ステップ（Claude Codeへの指示）
 1. package.json の作成と bin の設定。
 2. 上記ロジックを実装したスクリプトの作成。
 3. npm link（または同等の処理）を実行し、即座に zlog コマンドを有効化する。
 4. 正常に daily\_log.csv が生成・更新されることをテストする。
	 
## 💡 このツールが目指す「引き算」のゴール
 * **人間:** 「今何をしているか」を打つだけ。日時の確認や、Keepへのコピペ準備から解放される。
 * **PC:** 0.1秒で事実（時間）を記録。
 * **AI (Next Step):** 業務終了後にこのCSVを読み込み、カテゴリ体系（A1〜E0）を付与し、人間・Gemini用のプロトコル形式へ一括変換する。


# 🛠 雑ログ整理・アーカイブ（Step 2）追加実装指示書
## 1. 目的
zlog (Step 1) で蓄積された daily\_log.csv を読み込み、AI（Anthropic API）を用いて「カテゴリ付与」と「1行フォーマット整形」を行い、月別アーカイブファイルへ自動追記（ストック）する。

## 2. 技術要件
 * **使用API:** Anthropic SDK (@anthropic-ai/sdk)
 * **使用モデル:** claude-3-5-haiku-latest (デフォルト。設定で変更可能にすること)
 * **認証:** 環境変数 ANTHROPIC\_API\_KEY または設定ファイルから取得。

## 3. 実行コマンド
 * **コマンド:** zlog summarize (または zlog s)

## 4. 処理詳細
### ① データ入力と前処理
 * ~/zlog/daily\_log.csv から、**本日（実行日）の日付**のログを全件読み込む。
 * 各行の「開始時刻」と「終了時刻」の差分から、**所要時間（分）**をプログラム側で算出する。
### ② AIによる知的変換 (Prompting)
AIに対して、以下のプロトコルを適用して「カテゴリ付与」と「整形」を行わせる。
 * **カテゴリ体系:** A1-A6 (クライアントワーク), B1-B4 (自社), C1-C3 (社会), D1-D2 (私用/休憩), E0 (その他)
 * **1ログ1行の制約:** AIへの指示に「解説や前置きを一切含まず、1件につき必ず1行で出力せよ」という制約をかける。
### ③ 出力フォーマット (1ログ1行)
以下の形式を厳守する。
[YYYY-MM-DD] HH:MM-HH:MM | 案件名 | 作業内容 | カテゴリ（絵文字） | 所要：○分

## 5. 自動アーカイブ機能（最重要）
 * **保存先:** ~/zlog/archives/YYYY-MM.md (月ごとにファイルを自動作成・追記)
 * **追記ルール:**
   - ファイルが存在しない場合は新規作成。
   - 当日の見出し（## YYYY-MM-DD: 実績報告）がまだない場合、見出しを挿入。
   - AIが生成した「1ログ1行」のリストを、その日のセクションの末尾に追記する。
## 6. Gemini（秘書）への連携
 * 処理完了後、ターミナル上にアーカイブしたログの内容をそのまま出力する。
 * （ユーザーはこれをコピーして、Geminiとの振り返り対話に使用する）
 \* 
## 7. 設定（Configuration）
zlog config コマンド等で、以下の項目を永続化・変更可能にすること。
 * model: AIモデルの指定
 * api\_key: Anthropic APIキー
 * category\_protocol: カテゴリ定義（プロンプトに注入する内容）
